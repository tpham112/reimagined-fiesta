    Fire Tech Enrollment¶

    Year-Long Sections¶

    • Starting 2025SP, year-long sections are built for the term in which the section ends.
      • Example: A class starting in 2025SP but ending in 2026SP is built as a 2026SP course.

    BIDDLE (FAC 008)¶

    • If we have a Course Repetition Petition OR if the student has not previously taken the class: register the student. Override any restrictions encountered.
    • Save the SROS file.

    STAC: 1. Enter FRT in Notes/Dates. 2. Add a comment in Comments. 3. Save All.

    Once registration (and application loading if needed) is complete:

    • Scan and save the whole packet (Paper Packet, BRGN Output, SROS Output, Course Repetition Forms; combine through Adobe) to: M:\Instructor Grade Book\Term YYYY - Instructor Folders\00FTA

    Lifeguard ISA¶

    • Terry should only submit once, October of the following year.
    • He should check throughout the year for whoever needs applications, CAP forms, non-resident fixes, etc.
    • We would need to edit the Name Search file to narrow down who we can register.

    Steps¶

    1. Access CAP Forms¶

    • Receive enrollment request from coordinator.
    • Access the OneDrive link to retrieve CAP forms for the section.
    • Verify all necessary documentation is available.

    2. Batch Flag Students¶

    • Batch flag all students in the system to enable the Name Search to identify if they have already submitted applications.

    3. Run Name Search¶

    The Name Search returns: - Student ID numbers - Last enrolled term - Last term applied - Student type (Regular or CAPU) - CAPU form on file status (Yes/No)

    4. Non-Update Batch Registration¶

    • Run a non-update batch registration to identify which students are eligible — this test run shows feasibility without making changes.

    5. Cross-Reference with Master Sheet¶

    • Match batch registration results with students from the master sheet.
    • Highlight students who need to be registered and are eligible.

    6. Filter CAPU Students¶

    • Filter to show only CAPU students, then filter further to show only those with a CAPU form on file.

    7. Verify Eligibility¶

    • Check last term attended and last term applied for each CAPU student.
    • Confirm eligibility.
    • Highlight eligible students.

    8. Register Students¶

    • Filter to show only highlighted students.
    • Process batch registration for all highlighted students.

    9. Return Unregistered Students¶

    • Compile list of unhighlighted (unregistered) students.
    • Email the list to the coordinator (e.g., Terry).

    Coordinator Process for Next Enrollment Round¶

    1. Receive unregistered student list from specialist.
    2. Do not upload the standard blue and white Microsoft Form sheet.
    3. Use the running list of unregistered students provided.
    4. Add new students only from the blue and white Microsoft Form Excel (anyone who signed up after the previous submission).
    5. Upload the combined running list (previous unregistered + new students) for the next enrollment request.
