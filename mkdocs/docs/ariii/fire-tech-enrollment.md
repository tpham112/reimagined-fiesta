## Lifeguard ISA

- We would need to edit the Name Search file to narrow down who we can register.

### Steps

#### 1. Access CAP Forms
- Receive enrollment request from coordinator.
- Access the OneDrive link to retrieve CAP forms for the section.
- Verify all necessary documentation is available.

#### 2. Batch Flag Students
- Batch flag all students in the system to enable the Name Search to identify if they have already submitted applications.

#### 3. Run Name Search
The Name Search returns:
- Student ID numbers
- Last enrolled term
- Last term applied
- Student type (Regular or CAPU)
- CAPU form on file status (Yes/No)

#### 4. Non-Update Batch Registration
- Run a non-update batch registration to identify which students are eligible — this test run shows feasibility without making changes.

#### 5. Cross-Reference with Master Sheet
- Match batch registration results with students from the master sheet.
- **Highlight** students who need to be registered and are eligible.

#### 6. Filter CAPU Students
- Filter to show only CAPU students, then filter further to show only those with a CAPU form on file.

#### 7. Verify Eligibility
- Check last term attended and last term applied for each CAPU student.
- Confirm eligibility.
- **Highlight** eligible students.

#### 8. Register Students
- Filter to show only highlighted students.
- Process batch registration for all highlighted students.

#### 9. Return Unregistered Students
- Compile list of unhighlighted (unregistered) students.
- Email the list to the coordinator (e.g., Terry).
