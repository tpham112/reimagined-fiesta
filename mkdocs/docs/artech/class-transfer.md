# Class Transfer (AR Tech)

Applies to both individual transfers and division-requested bulk transfers for class cancellations.

!!! warning
    Must be **same length to same length** (e.g., 8-week to 8-week; 16-week to 16-week).

## Class Transfer Before Census

**RGN:**

1. Enter Student ID and detail into the old course.
2. Update Status to **`D`** (Dropped).
3. Backdate to before Census so the student does not get double-charged.
4. Update Status Reason to **`TR`**.
5. Click Save.
6. Go to the last line and type in the new section.
7. Detail into the new section.
8. Update Status Reason to **`TR`**.
9. Click **Save All**.

**STAC:**

1. Detail into the dropped section.
2. Add **`TR`** code to the dropped section — note the time.
3. Click Save.
4. Detail into the new section.
5. Add **`TR`** code to the added section — change add time to 1 minute after the class drop.

**If online course:** Email DistanceEd@sac.edu to move Canvas submissions from old section to new section.

---

## Class Transfer After Census

**RGSD:** Backdate system to the Friday before Census date, then follow all steps above.

!!! danger
    Don't forget to **return RGSD to current date** after processing!

---

## Error — Reversing a Transfer

**STAC:** Note the class section that was dropped in error.

**RGN:** Drop the added section and re-register in the prior section.

**STAC:**

1. Add **`ER`** (Error) code to the newly dropped/wrong section and add a comment explaining why it was added then dropped. Be specific. Timestamp.
2. Drill into the class that was added back — delete the add and drop line so only the original registration information remains.

## Waitlist Rollover — By Division Request

### Summary

1. Division provides Waitlist Roster (AM118 report).
2. Ensure there is space in the desired section.
3. Batch Registration — verify eligibility first (run on **No Update** first).
4. Drop students from their old waitlists.

### Steps

**SWLM or Query — Get students in requested sections:**

1. Go to the section of students to be moved.
2. Export the Excel sheet.

**SLED:** Put students in SLED list.

**SCBL:**

1. Name your Section Block (will be used in BRGN).
2. Enter Term/Start/End Date.
3. Enter course name and section number.

**BRGN:**

| Field | Value |
|-------|-------|
| Section Block | Your SCBL name |
| Reg User Group | `WEBREG` |
| Date | Today's date |
| Update | Test with **N** first, then **Y** |

Remove any students with errors from the list before running with Y.

**SWLM:** Drop students who were registered from their waitlist.

**Final Steps:**

1. Send email with Excel sheet of registered/non-registered students.
2. File email and processed waitlist roster to student records in Laserfiche.

## Waitlist Rollover WLAV — After Term Start (Manual Daily)

The savelist is generated by ITS in SLCR and run in the background nightly.

- **WLAV.SL** — Use if the session you are running for has already started. Holds students that need to be rolled over for sections within that term that have not yet started.

If the Division needs an individual WL rollover for one section during the day:
- Remove the Savelist name.
- Add the section number.
- Run.

## Batch Transfer

### Before Section Start
If the department is going to cancel the section anyway — have them just cancel the section, then batch register once students are deregistered from the cancellation.

### After Section Start — Run Batch Transfer Process

**Batch Drops:**

SLED — Load Course.Section.IDs via DREG Query.

DREG — Run drop.

**Batch Enroll:**

SLED — New SLED list with true Student IDs.

SCBL — Use previously created list. Enter class name and section number. *(Multiple sections can be enrolled at the same time only if exact same roster AND same section dates.)*

BRGN:

| Field | Value |
|-------|-------|
| Section Blocks | Name used in SCBL |
| RG User Group | `ARSTAFF` |
| Date | Today (backdate for CJ if needed) |
| Update | **No** first to test — essential to get registration errors |
| Saved List Name | Name used in SLED |

Critical errors requiring student removal from SLED:
- Require Overload Petition
- Repeat — does not meet requirements
- Does not meet prereq
- Time conflict

Run again with **Update: Yes** to actually enroll.

**STAC:** Add **`TR`** coding to each record. Check that enroll time is after drop time.
